# AI MIDI 생성 시스템 PRD

## Overview
자연어 프롬프트를 통해 트랙별, 송 파트별로 세분화된 MIDI 파일을 생성하는 AI 시스템입니다. 복잡한 오디오 처리 대신 **고품질 MIDI 생성**에 집중하여 실용적이고 구현 가능한 솔루션을 제공합니다.

**문제점**: 기존 AI 음악 생성 도구들은 너무 복잡하거나 세부 제어가 어렵습니다.
**해결책**: 트랙별, 파트별로 정확하게 제어 가능한 MIDI 생성으로 실용적인 작곡 도구 제공
**대상 사용자**: 개발자 본인 (개인적인 송라이팅 및 음악 제작, 완전한 커스터마이징 가능)

## Core Features

### 1. 트랙별 + 송 파트별 세분화 생성 ⭐ 핵심 기능
- **기능**: 특정 악기의 특정 송 파트만 독립적으로 생성
- **예시**: 
  - "인트로용 103BPM 어두운 붐뱁 드럼 트랙"
  - "브릿지용 기타 솔로 트랙"  
  - "코러스용 신스 리드 멜로디"
  - "벌스용 베이스 라인"
- **중요성**: 실제 작곡 워크플로우에 맞는 세밀한 제어
- **구현**: 악기 타입 + 송 파트 + 음악적 특성을 조합한 생성

### 2. 모듈식 4마디 스니펫 생성 🧩 핵심 작곡 이념
- **기능**: 4마디 단위의 레고 블록식 MIDI 스니펫 생성
- **철학**: "음악은 4마디 블록들의 조합" - 현실적이고 실용적 작곡 접근
- **모듈 타입**:
  ```
  인트로 모듈: 분위기 설정, 점진적 빌드업
  벌스 모듈: 기본 멜로디, 스토리텔링 중심  
  프리코러스 모듈: 에너지 상승, 코러스 준비
  코러스 모듈: 임팩트, 기억에 남는 훅
  브릿지 모듈: 대조, 새로운 관점
  아웃트로 모듈: 마무리, 페이드아웃
  필인 모듈: 섹션 간 연결, 1-2마디
  ```
- **중요성**: 
  - 실제 작곡가들의 사고 방식과 일치
  - 무한한 조합 가능성 (레고 블록처럼)
  - 부분적 수정과 교체 용이
  - 협업과 아이디어 공유에 최적
- **구현**: 각 모듈별 특화된 AI 생성 + 모듈 간 조화 시스템

### 3. 계층적 장르 시스템 (3단계 + 크로스 장르 조합)
- **기능**: 메인 장르 → 세부 장르 → 스타일 변형으로 정밀한 스타일 지정
- **3단계 구조**:
  ```
  힙합 → 트랩 → 멜로딕 트랩 / 하드 트랩 / 로파이 트랩
       → 붐뱁 → 올드스쿨 붐뱁 / 모던 붐뱁 / 재즈 붐뱁
       → 트릴 → 클래식 트릴 / 멜로딕 트릴 / 다크 트릴
  
  락 → J락 → 시부야계 / 비주얼계 / 얼터너티브 J락
     → 메탈코어 → 멜로딕 메탈코어 / 프로그레시브 메탈코어 / 브레이크다운 메탈코어
     → 인디락 → 드림팝 / 매스락 / 포스트락
  
  일렉트로닉 → 하우스 → 딥하우스 / 테크하우스 / 프로그레시브 하우스
            → 테크노 → 미니멀 테크노 / 하드 테크노 / 멜로딕 테크노
            → 트랜스 → 업리프팅 트랜스 / 사이키델릭 트랜스 / 프로그레시브 트랜스
  ```
- **크로스 장르 조합**: 
  ```
  "헤비메탈 기타 리프 + 트랩 드럼" → Metal-Trap Fusion
  "재즈 코드 진행 + 로파이 힙합 드럼" → Jazz-LoFi Hybrid
  "K팝 멜로디 + UK 드릴 베이스" → K-Drill Style
  ```
- **중요성**: 
  - 무한한 음악적 실험 가능성
  - 새로운 장르 창조 및 개인 스타일 개발
  - 모듈별 독립적 장르 적용으로 창의성 극대화
- **구현**: 장르별 특화된 MIDI 패턴 데이터베이스 + 크로스 장르 조합 알고리즘

### 4. 유연한 언어모델 아키텍처
- **기능**: 다양한 LLM을 쉽게 교체할 수 있는 플러그인 구조
- **지원 모델**: 
  - Claude (Anthropic)
  - EXAONE 3.5 (LG AI Research)
  - GPT-4 (OpenAI)
  - 커스텀 모델 (향후 확장)
- **중요성**: 최신 모델 활용 + 비용 최적화 + 성능 비교
- **구현**: Provider 패턴으로 LLM 추상화

### 5. 스마트 프롬프트 엔지니어링
- **기능**: 자연어를 구조화된 음악 파라미터로 변환
- **입력 처리**: 
  - 명시적 파라미터: "120 BPM", "C major", "4/4 박자"
  - 암시적 표현: "밝고 경쾌한", "어두운 분위기", "여름 느낌"
- **출력 형식**: JSON 구조화된 음악 파라미터
- **중요성**: 자연스러운 사용자 경험의 핵심

### 6. 모듈식 MIDI 생성 엔진
- **기능**: 악기별 특화된 MIDI 생성 모듈
- **지원 악기**:
  - **드럼**: 킥, 스네어, 하이햇, 심벌, 퍼커션
  - **베이스**: 일렉트릭, 어쿠스틱, 신스 베이스
  - **화성 악기**: 피아노, 기타, 신스 패드/리드
  - **멜로디**: 보컬 라인, 솔로 악기, 리드 멜로디
- **중요성**: 각 악기의 특성에 맞는 고품질 생성
- **구현**: 악기별 전용 생성 알고리즘 + 음악 이론 적용

### 7. 지능형 데이터 관리 시스템 🔧 개발자 전용
- **기능**: 학습 데이터셋의 수동 검증, 수정, 보강 시스템
- **핵심 기능**:
  ```
  데이터 검증 워크플로우:
  1. 자동 수집된 MIDI 데이터 검토
  2. 아티스트, 국가, 곡명 정보 확인/수정
  3. 장르 분류 3단계 검증 (메인→세부→스타일)
  4. 개인적 설명(description) 추가
  5. 품질 점수 (1-10) 부여
  6. 검증 완료 마크
  ```
- **수동 관리 도구**:
  - **웹 기반 데이터 에디터**: 직관적인 수정 인터페이스
  - **배치 수정**: 여러 데이터 동시 업데이트
  - **검증 진행률**: 전체 데이터셋 검증 상태 추적
  - **품질 필터링**: 고품질 데이터만 학습에 사용
- **중요성**: 
  - 개인 취향 반영된 고품질 데이터셋 구축
  - 특정 아티스트 스타일 정확도 향상
  - 국가별 장르 변형 세밀한 분류
- **구현**: 관리자 웹 인터페이스 + 데이터 검증 API

### 8. 아티스트 기반 생성 & 곡 제목 추천 🎵
- **기능**: 특정 아티스트 스타일 모방 + 자동 곡 제목 생성
- **아티스트 기반 생성**:
  ```
  "ONE OK ROCK 스타일 J락 기타 리프, 코러스용"
  "지코 스타일 트랩 랩 플로우, 벌스용"  
  "선미 스타일 K팝 신스 멜로디, 후크용"
  ```
- **곡 제목 추천 시스템**:
  - **분위기 기반**: "어두운 트랩 비트" → "Midnight Hustle", "Shadow Games"
  - **장르 기반**: "J락 발라드" → "桜の記憶", "마지막 약속"
  - **크로스 장르**: "Metal-Trap" → "Steel Trap", "Iron Beats"
- **국가별 특화**:
  - **한국**: K팝, K힙합, 트로트 변형
  - **일본**: J락, 시부야계, 비주얼계
  - **영국**: UK 드릴, 브릿팝, 더브스텝
  - **미국**: 남부 힙합, 웨스트코스트, 트랩
- **중요성**: 
  - 개인적 음악 취향 정확히 반영
  - 곡 완성도 향상 (제목까지 제안)
  - 창작 영감 제공
- **구현**: 아티스트 스타일 DB + 제목 생성 패턴 엔진

## User Experience

### Primary User Flow: 레고 블록식 작곡

#### 1. 자연어 입력 → 구조화된 결과물 출력

**입력 예시**:
```
"어두운 분위기의 붐뱁 드럼, 인트로용, 85BPM, 4마디"
```

**AI 분석 결과**:
```json
{
  "module_type": "intro",
  "instruments": ["drums"],
  "genre": "hip_hop",
  "sub_genre": "boom_bap", 
  "style_variant": "old_school_boom_bap",
  "mood": "dark",
  "tempo": 85,
  "key": "A_minor",
  "duration": "4_bars",
  "complexity": "medium",
  "energy_level": "low"
}
```

**최종 결과물 구조**:
```
📁 generated_modules/
├── 📄 dark_boombap_intro_85bpm.mid          # 메인 MIDI 파일
├── 📄 dark_boombap_intro_85bpm_meta.json    # 메타데이터
├── 📄 dark_boombap_intro_85bpm_preview.wav  # 미리보기 오디오 (선택적)
└── 📄 dark_boombap_intro_85bpm_analysis.txt # AI 분석 내용
```

#### 2. 4마디 스니펫 직접 관리 시스템 🎛️

**A. 스니펫 라이브러리 구조**:
```
📁 my_snippets/
├── 📁 intro_modules/
│   ├── dark_boombap_intro_85bpm.mid
│   ├── bright_trap_intro_120bpm.mid
│   └── jrock_emotional_intro_110bpm.mid
├── 📁 verse_modules/
│   ├── zico_style_trap_verse_140bpm.mid
│   ├── lofi_guitar_verse_90bpm.mid
│   └── metal_bass_verse_130bpm.mid
├── 📁 chorus_modules/
│   ├── kpop_synth_chorus_128bpm.mid
│   ├── jrock_guitar_chorus_120bpm.mid
│   └── metal_trap_fusion_chorus_150bpm.mid
└── 📁 custom_variations/
    ├── dark_boombap_intro_85bpm_v2.mid      # 사용자 수정 버전
    ├── dark_boombap_intro_85bpm_v3.mid      # 또 다른 변형
    └── dark_boombap_intro_85bpm_hybrid.mid  # 다른 스니펫과 조합
```

**B. 스니펫 확인 및 교체 워크플로우**:

1. **즉시 미리보기**:
   ```
   생성 완료 → 자동 미리보기 재생 → 만족도 확인
   ```

2. **세부 편집 옵션**:
   ```
   만족하지 않을 경우:
   ├── "다시 생성" (같은 파라미터, 다른 결과)
   ├── "변형 생성" (약간의 파라미터 조정)
   ├── "수동 편집" (DAW에서 직접 수정)
   └── "유사 스니펫 추천" (라이브러리에서 비슷한 것 찾기)
   ```

3. **스니펫 관리 인터페이스**:
   ```
   📱 웹 기반 스니펫 매니저:
   
   [🎵 dark_boombap_intro_85bpm.mid] [▶️재생] [✏️편집] [🔄변형] [❌삭제]
   ├── 생성일: 2024-01-15 14:30
   ├── 사용 횟수: 3회
   ├── 평점: ⭐⭐⭐⭐☆ (4/5)
   ├── 태그: #어두운 #붐뱁 #인트로 #85BPM
   └── 메모: "좀 더 킥이 강했으면 좋겠음"
   
   [변형 버전들]
   ├── v2: 킥 강화 버전 ⭐⭐⭐⭐⭐
   ├── v3: 하이햇 추가 버전 ⭐⭐⭐☆☆  
   └── hybrid: 트랩 요소 추가 ⭐⭐⭐⭐☆
   ```

### 예상 소요 시간: 
- **단일 모듈**: 30초 - 1분
- **전체 곡 구성**: 5-10분 (모듈 조립 포함)

### 고급 워크플로우: 모듈 기반 작곡

#### 1. 모듈 라이브러리 구축
- **개인 스타일 모듈**: 자주 사용하는 패턴들
- **장르별 모듈 컬렉션**: 힙합, 락, 일렉트로닉 등
- **무드별 모듈 세트**: 어두운, 밝은, 에너지틱한 등

#### 2. 모듈 변형 및 발전
```
기본 벌스 모듈 → 변형 생성:
- "같은 코드 진행, 더 복잡한 리듬"
- "같은 멜로디, 다른 악기로"  
- "에너지 레벨 20% 상승"
```

#### 3. 모듈 간 자동 조화
- **키 호환성**: 자동 조 변환
- **템포 매칭**: BPM 자동 조정
- **에너지 곡선**: 자연스러운 섹션 전환

#### 4. 협업 및 공유
- **모듈 교환**: 다른 프로듀서와 모듈 공유
- **스타일 학습**: 좋아하는 아티스트 모듈 분석
- **레시피 저장**: 성공한 모듈 조합 패턴 기록

## Technical Specifications

### AI Models & APIs:
- **Claude 3.5 Sonnet**: 고품질 프롬프트 분석
- **EXAONE 3.5**: 한국어 특화 + 비용 효율성
- **Custom MIDI Models**: 악기별 특화 생성 (향후)

### MIDI Processing:
- **라이브러리**: `mido`, `pretty_midi`, `music21`
- **포맷**: Standard MIDI File (SMF) Format 1
- **해상도**: 480 ticks per quarter note

### Output Formats:
- **MIDI Files**: `.mid` (DAW 호환)
- **메타데이터**: JSON sidecar files
- **프로젝트**: Ableton Live `.als` (선택적)

### Integration:
- **DAW 호환성**: Ableton Live, Logic Pro, FL Studio
- **플러그인**: VST/AU 자동 매핑 (향후)
- **클라우드**: AWS S3 저장, CDN 배포
